Using Clone()

You are able to make a deep clone of every generated TransferObject, by calling 'clone()' on it.

This cloned object is outside of any caching that is currently in use within Transfer, which means that any changes that are done to this object, and are not saved, are only particlur to the request that they are currently in.

It should be noted that Transfer.save() and transfer.update() on a clone object will update the object currently in cache to the state of the saved object

Caveat on cloned objects

Cloned objects should not be added as compsites to objects that are not clones. This can cause indeterminate results.
